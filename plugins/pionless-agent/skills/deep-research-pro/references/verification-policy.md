# Verification Policy

## Required support

- Major claims need at least **three independent supporting sources** when feasible (stricter than the standard tier).
- Numeric claims, timelines, version claims, legal claims, and benchmark claims **must** be checked directly at the source.
- Predictive or speculative claims must cite who is saying them; do not state as fact.

## Independence

Three sources count as independent only if none of them derive from the same underlying report. Three outlets quoting the same press release are one source, not three. Track the lineage explicitly when in doubt.

## Conflicts

Surface, do not smooth.

- Note every disagreement explicitly in the evolving report's `Contradictions` section.
- Explain which source is closer to primary and why.
- Attempt to resolve via a **third source** that has access to the underlying evidence.
- If unresolvable, document both positions and which one current evidence favors.

## Single-source fallback

If only one source is available for a major claim within the workflow, this is unusual at the pro tier. Mark `confidence: low — single source only` and explicitly call it out as a research gap requiring further work.

## Dedicated contradiction pass

Before finalizing, run a separate contradiction-seeking pass that only tries to disprove the current thesis. The completion gate cannot pass without it. See `references/contradiction-seeking-pass.md`.

## Weakness disclosure

If evidence is weak, say so. Do not smooth over uncertainty in the synthesis. The report's `Open Risks & Unknowns` and `Contradictions & Contested Claims` sections are the right places to record gaps.
